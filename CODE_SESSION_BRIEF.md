# 代码会话协作说明

每次开启新的代码会话，都先读取本文件和 `XIANGDONGQU_PLAN.md`。

## 项目

项目名称：向东渠事迹数字馆新版原型

定位：参考旧案例主题，做一个现代化数字展馆 CMS。不是复刻旧网站，也不是改旧项目。目标是做一个能演示、能管理内容、后续可扩展上线的全栈原型。

## 协作方式

当前项目拆成两个会话配合：

- 计划会话：负责需求分析、阶段拆分、判断方向。
- 代码会话：负责按阶段实现、运行、检查、汇报文件和结果。

代码会话不要自作主张一次性做完所有阶段。每次只完成当前指定阶段。

## 每次代码会话开始时必须做

1. 读取 `C:/Users/admin/Documents/New project 13/CODE_SESSION_BRIEF.md`
2. 读取 `C:/Users/admin/Documents/New project 13/XIANGDONGQU_PLAN.md`
3. 明确当前只做用户指定阶段
4. 实现完成后运行检查
5. 汇报：
   - 本地访问地址
   - 启动命令
   - 改了哪些文件
   - 是否有报错
   - 下一步建议，但不要直接进入下一阶段，除非用户明确要求

## 当前阶段节奏

1. 阶段 1：搭建 Next.js + TypeScript + Tailwind 项目和前台页面骨架
2. 阶段 2：完善前台展示和 mock 数据
3. 阶段 3：整理内容结构
4. 阶段 4：后台管理界面
5. 阶段 5：API 和数据库
6. 阶段 6：素材上传和展示
7. 阶段 7：部署和演示

## 当前优先任务

现在优先做阶段 1。

阶段 1 只做：

- 搭建项目
- 前台页面骨架
- mock 数据
- 基础导航
- 移动端适配
- 能通过 `npm run dev` 运行

阶段 1 不做：

- 数据库
- 登录
- 文件上传
- 真后台 CRUD
- 部署

## 阶段 1 页面要求

前台至少包含：

- 首页
- 数字导览
- 历史资料
- 精神内涵
- 时代价值
- 资讯动态
- 视频展播
- 后台入口

首页要像“数字展馆”或“专题馆”，不要做成普通博客、普通公司官网或营销落地页。

## 风格方向

关键词：

- 红色文化
- 水利工程
- 历史资料
- 数字展馆
- 稳重
- 现代
- 可维护

不要过度花哨，不要做成游戏风、二次元风、纯科技蓝大屏风。

## 给老板看的价值点

这个原型要体现：

- 不是静态页面
- 后续可以接后台内容管理
- 以后可以放照片、视频、采访、文献资料
- 如果效果好，可以继续扩展成正式系统

## 常用启动检查

实现完成后优先运行：

```bash
npm run dev
```

如果项目有 lint/build，也可以运行：

```bash
npm run lint
npm run build
```

但不要为了修无关问题大范围重构。
