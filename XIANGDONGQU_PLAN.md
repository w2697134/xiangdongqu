# 向东渠事迹数字馆新版原型计划

## 项目定位

项目名称：向东渠事迹数字馆新版原型

目标：参考旧案例的主题和内容结构，做一个现代化数字展馆 CMS。前台用于展示向东渠历史、精神、资料、视频、VR 导览；后台用于管理文章、图片、视频、文献、首页推荐等内容。

这个项目不是复刻旧网站，也不是直接改旧项目，而是一个可扩展的新版全栈原型。如果后续效果好，可以继续扩展成正式网站、小程序后台或专题展示系统。

## 给老板看的核心说法

我参考之前“向东渠事迹数字馆”的主题，做了一个新版数字展馆原型。它不是静态页面，而是预留了后台内容管理。以后如果有新的照片、视频、采访、文献资料，可以直接从后台上传和更新，前台会自动展示。后续如果需要，也可以继续扩展成正式网站或小程序后台。

## 交付原则

正式给老板交付/演示只放在最终阶段。阶段 1 到阶段 6 都属于内部推进、阶段验收和能力补齐，不作为最终交付版本。

最终交付前必须至少满足：

- 前台页面完整、视觉稳定、移动端可用
- 后台能新增、编辑、删除核心内容
- 图片、视频链接、文献资料、VR 导览等可从后台维护
- 前台能读取后台或数据库内容并自动展示
- 示例数据和待上传位置表达清楚
- `npm run lint`、`npm run build` 和浏览器检查通过
- README 或交付说明写清楚项目定位、运行方式、功能模块和后续扩展方向

## 总体节奏

1. 先做前台展示
2. 再做后台界面
3. 再接 API 和数据库
4. 再做上传功能
5. 最后部署演示

不要一开始就追求全功能。先做出老板能看懂、能点、能感受到价值的版本。

## 阶段 1：项目定位和页面骨架

你要干啥：

先明确这个项目不是复刻旧网站，而是新版数字展馆原型。先不要急着做数据库、登录、上传，先让项目看起来像一个完整产品。

让另一个会话干啥：

```text
请在当前项目中搭建一个 Next.js + TypeScript + Tailwind CSS 的全栈项目，主题是“向东渠事迹数字馆新版原型”。先不要做复杂数据库，先完成前台页面结构和假数据展示。要求首页不是普通官网，而是数字展馆风格，包含首页、数字导览、历史资料、精神内涵、时代价值、资讯动态、视频展播、后台入口。
```

## 阶段 2：前台展示先做漂亮

你要干啥：

先把老板能看到的东西做好。老板第一眼看的不是数据库，而是“这个东西像不像正式项目”。

让另一个会话干啥：

```text
请先实现前台页面。页面包括：
1. 首页：主题视觉、向东渠简介、核心入口、推荐内容
2. 数字导览：VR导览、展厅导览、路线导览入口
3. 历史资料：大事记、旧文献、老照片、老物件
4. 精神内涵：勇于担当、团结协作、攻坚克难、开拓创新
5. 时代价值：人物风采、先进事迹、研究资料
6. 资讯动态：新闻列表和新闻详情
7. 视频展播：视频卡片、封面、标题、简介
先用本地 mock 数据，页面要完整、现代、移动端适配。
```

## 阶段 3：把内容结构设计清楚

你要干啥：

确定以后要放哪些内容，尤其是老板拍摄的照片、视频、采访、文献资料。

让另一个会话干啥：

```text
请整理项目的 mock 数据结构，前台所有页面都从统一数据结构读取，后面接数据库时不用大改。

至少包含这些内容类型：
1. 文章/新闻
2. 历史资料
3. 老照片
4. 老物件
5. 旧文献
6. 视频
7. VR导览
8. 首页推荐位
9. 栏目分类
```

## 阶段 4：做后台管理界面

你要干啥：

这一步很关键。要让老板看到：“以后你拍了东西，可以从后台传上去更新。”

让另一个会话干啥：

```text
现在给项目增加后台管理页面。后台路径使用 /admin。先做简单登录页和后台布局，然后实现这些管理模块：
1. 文章管理
2. 图片/资料管理
3. 视频管理
4. VR导览链接管理
5. 首页推荐位管理
6. 栏目管理

目前可以先用 mock 数据或本地状态完成新增、编辑、删除的界面体验，重点是让老板能看懂“以后可以在后台更新内容”。
```

## 阶段 5：接 API 和数据库

你要干啥：

当前台和后台都像样之后，再做真正全栈。不要一开始就上数据库，否则容易卡住。

让另一个会话干啥：

```text
请把项目改成真正的全栈结构，使用 Prisma + SQLite 作为本地数据库。设计数据模型：
Category 栏目
Article 文章/新闻/专题
Media 图片、文献、资料
Video 视频
VrTour VR导览链接
Banner 首页推荐位
AdminUser 管理员

并实现基础 API：
/api/articles
/api/categories
/api/media
/api/videos
/api/vr-tours
/api/banners
/api/upload

后台管理页面要调用这些 API 完成新增、编辑、删除、列表展示。前台页面要从 API 或数据库读取内容。
```

## 阶段 6：做素材上传和展示

你要干啥：

重点围绕“老板拍摄的东西以后能不能放进去”。这是项目最有价值的地方。

让另一个会话干啥：

```text
请实现素材上传和展示能力：
1. 图片上传
2. 视频链接管理
3. 本地视频信息管理
4. 文献资料链接管理
5. 首页推荐内容选择

上传或新增后，后台能看到，前台对应栏目能自动展示。
```

## 阶段 7：部署和演示

你要干啥：

准备给老板看的版本。演示重点是完整流程：后台新增内容，前台立刻展示。

让另一个会话干啥：

```text
请帮我完善项目的演示体验：
1. 首页内容更像正式数字展馆
2. 后台有几条示例数据
3. 前台能展示后台新增的内容
4. 补充 README，说明项目定位、功能模块、技术栈、如何运行、后续可扩展方向
5. 确保 npm run dev 可以正常运行
```

## 注意事项

- 不要说“我用 GPT 生成了网页”，要说“我用 AI 辅助完成需求分析、页面设计、前后端开发和后台内容管理原型”。
- 不要一开始承诺能替换正式系统，可以说“后续如果合适，可以继续扩展成正式系统”。
- 老板更容易理解后台上传、前台展示这种能力，所以一定要做后台。
- 先做能演示的版本，再做复杂功能。
