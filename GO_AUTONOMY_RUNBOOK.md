# Go 自主推进规则

用途：当用户说“Go”“继续”“你自主推进”时，总控会话按本文件执行，不需要每一步都反复问用户。

## 当前长期会话

- 代码会话：`019e2aef-d102-73d1-864c-67fa60b23dda`
- UI / 图片设计会话：`019e2af5-de8e-7353-906d-7c39a4c61220`
- 视觉验收与拼装会话：`019e2b6e-7d90-7ad2-86f5-283a2706533d`

更多 ID 见 `AGENT_SESSION_IDS.md`。

## 总控会话职责

总控会话负责：

- 判断阶段目标
- 给代码会话派任务
- 给 UI 会话派视觉资产任务
- 验收代码和图片
- 更新项目文档
- 运行 lint/build/本地页面检查
- 必要时开短期子智能体做只读复核
- 用完短期子智能体后关闭
- 接收视觉验收与拼装会话的细调结果，并做最终验收判断

总控会话不应该：

- 长期把大量图片堆进主聊天
- 让代码会话一次性跨太多阶段
- 未经用户同意直接上数据库、登录、上传、部署
- 用生成图冒充真实历史照片、真实文献或老板拍摄素材
- 把阶段 1 到阶段 6 的内部验收版本说成可以最终交付给老板

## 长期会话职责边界

- UI / 图片设计会话：负责视觉方向和分层资产生成。生图必须派 worker 落盘。
- 视觉验收与拼装会话：负责资产放入页面后的细调、拼装和阶段视觉验收。
- 代码会话：负责实现页面、动效、路由、数据和工程质量。
- 总控会话：负责需求判断、跨会话调度、最终验收和阶段推进。

视觉验收与拼装会话的硬性规则：

- 不能生成任何图片，不能调用图片生成工具。
- 只有 UI / 图片设计会话可以调度生图。
- 必须看拼装后的页面效果，不只看单张资产。
- 必须截图桌面和移动端关键状态。
- 如果发现资产不合格，反馈给总控，由总控要求 UI 会话重做。
- 重做资产放回页面后，必须再次拼装验收。
- 不能因为“资产已经重做”就跳过二次验收。
- 视觉验收通过不等于最终通过，最终验收由总控会话决定。

## 用户说 Go 后的默认推进顺序

### 1. 收尾当前代码任务

先恢复或等待代码会话：

```text
019e2aef-d102-73d1-864c-67fa60b23dda
```

检查它是否完成当前阶段 1 返工：

- 是否切换 `/generated/...` 正式视觉资产
- 是否补充“待放/待上传”占位表达
- 是否修复移动端首页标题拥挤
- 是否通过 `npm run lint`
- 是否通过 `npm run build`

如果没完成，继续让代码会话完成，不直接进入下一阶段。

### 2. 阶段 1 最终验收

总控会话本地检查：

- `npm run lint`
- `npm run build`
- `http://localhost:3000`
- 首页、数字导览、历史资料、后台入口
- 移动端首页和移动端菜单

必要时使用短期 QA 子智能体做只读复核。复核完成后关闭该子智能体。

### 3. 如果阶段 1 通过，进入阶段 2

阶段 2 目标：

- 让前台更像正式数字展馆
- 用第一批正式生成图提升首页、导览、历史资料视觉
- 完善 mock 数据展示
- 补“待放/待上传”真实素材位置
- 优化移动端观感
- 仍然不做数据库、登录、上传、真后台 CRUD

### 4. 同步派 UI 会话做第二批资产

恢复 UI 会话：

```text
019e2af5-de8e-7353-906d-7c39a4c61220
```

让 UI 会话作为调度者，必须派子智能体/worker 生图、落盘、更新 manifest；UI 主会话不要自己把图堆聊天里。

第二批建议资产：

- `cover-spirit.png`
- `cover-era-value.png`
- `cover-video.png`
- `cover-news.png`

后续再做：

- Logo 概念
- 老照片占位图
- 老物件占位图
- 旧文献占位图
- 后台登录页背景

每批生成后必须：

- 保存到 `design-assets/generated`
- 同步到 `public/generated`
- 更新 `DESIGN_ASSET_MANIFEST.md`
- 汇报路径和用途

### 5. 给代码会话派阶段 2 任务

阶段 2 默认任务：

- 首页第一屏使用正式主视觉
- 首页核心入口卡片使用生成图封面，不只是文字卡片
- 历史资料页使用档案馆风格
- 数字导览页使用数字导览封面
- 视频展播、资讯动态、精神内涵、时代价值做得更完整
- 明确所有未来真实素材位置的占位状态
- 优化移动端标题、卡片间距和导航体验

阶段 2 不做：

- Prisma
- SQLite
- API
- 登录
- 上传
- 真实后台 CRUD
- 部署

### 6. 阶段 2 验收

验收重点：

- 老板第一眼能看懂这是“向东渠事迹数字馆”
- 不像普通博客或模板站
- 生成图片没有冒充真实史料
- 待放位置清楚
- 移动端无明显拥挤、遮挡、文字溢出
- lint/build 通过
- 本地页面可访问

## 自主程度边界

用户说 Go 后，总控可以自主做到：

- 指挥代码会话完成当前阶段返工
- 指挥 UI 会话生成下一批视觉资产
- 本地运行检查
- 开短期子智能体做只读验收
- 关闭用完的短期子智能体
- 更新 Markdown 文档
- 给出下一阶段明确任务

用户未明确同意前，不自主做：

- 正式数据库接入
- 登录鉴权
- 上传文件
- 部署上线
- 购买服务或使用付费外部资源
- 冒充真实资料的数据迁移
- 大范围重构

## 对外交付边界

阶段 1 到阶段 6 只做内部验收，不建议正式交给老板。

只有阶段 7 完成后，才按最终交付口径汇报。阶段 7 至少要包含：

- 前台完整展示
- 后台可维护核心内容
- API 和数据库连通
- 素材上传或素材链接管理可用
- 前台能展示后台新增内容
- 本地或线上演示环境稳定
- 交付说明文档齐全

## 汇报格式

每次自主推进一轮后，最终汇报：

```text
当前阶段：
完成事项：
代码会话状态：
UI 会话状态：
验收结果：
发现的问题：
下一步建议：
```

如果出现阻塞，直接说明阻塞点和最小解决方案。
