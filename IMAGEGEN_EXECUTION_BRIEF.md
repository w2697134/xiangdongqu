# ImageGen 执行说明

这个文件给设计图片会话或图片生成子智能体使用，目的是避免会话只检查目录、只写提示词，却没有真正调用图片生成工具。

硬性方式：设计 UI 会话只作为调度者，必须派子智能体/worker 执行图片生成和文件落地。子智能体负责实际生成图片、保存文件、更新 manifest；设计 UI 会话只接收路径和结果，不要把生成图直接堆在主聊天框。

## 必须先确认能力

开始生成前，执行生成的会话或子智能体必须明确说明：

- 是否可以调用 `image_gen` / 图片生成工具
- 如果可以，马上调用图片生成工具
- 如果不可以，停止并说明“当前会话没有图片生成工具”，不要假装生成、不要只创建空文件、不要用占位图冒充生成图

## 正确流程

1. 读取 `DESIGN_ASSET_MANIFEST.md`
2. 确认要生成的文件名和提示词
3. 调用图片生成工具生成图片
4. 找到生成后的真实图片文件
5. 保存或复制到：

```text
C:/Users/admin/Documents/New project 13/design-assets/generated
```

6. 如果代码项目目录存在，也复制到：

```text
C:/Users/admin/Documents/New project 13/public/generated
```

7. 更新 `DESIGN_ASSET_MANIFEST.md`
8. 汇报真实文件路径和代码引用路径

## 不算完成的行为

这些都不算生成完成：

- 只检查 `Test-Path`
- 只创建目录
- 只输出提示词
- 只说“我会生成”
- 只生成空白占位文件
- 只把路径写进 manifest 但没有真实图片

## 如果工具无法直接保存到指定目录

内置图片生成工具通常会先把图保存到默认生成目录。正确做法是：

1. 先调用图片生成工具
2. 找到生成图文件
3. 再复制到项目目录

不要在生成前假设图片已经在项目目录里。


每次最多生成 1 到 3 张。生成后不要在聊天里堆大量图片，只汇报：

- 用途
- 是否已更新 manifest

## 如果没有图片生成工具

如果当前会话或子智能体没有 `image_gen` 或图片生成能力，直接回复：

```text
当前会话没有可用的图片生成工具，我不能真正生成图片。请换到有 image_gen 工具的会话，或让总控会话生成。
```

不要继续假装执行。
