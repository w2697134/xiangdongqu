# UI 视觉标准

这个文件记录“向东渠事迹数字馆新版原型”的高质量 UI 和图片资产要求。

## 核心原则

这个项目要按作品级标准做，不是普通练习页面。

目标不是“能跑”，而是让老板看到后觉得：

- 这个方向认真
- 视觉完成度高
- 不是模板站
- 后续真的可以放照片、视频、采访、文献
- 有机会继续扩展成正式系统

## 生成图片的使用原则

只要生成图片能明显提升效果，就应该生成。

优先使用生成图片的地方：

- 首页主视觉
- 栏目封面
- 视频封面
- 资料库封面
- 空状态插图
- 展馆氛围图
- 后台概览页视觉
- 分享封面图

谨慎使用生成图片的地方：

- Logo
- 图标
- 真实历史照片
- 文献扫描件
- 带文字的海报

原因：

- Logo 更适合做成可缩放 SVG。可以先用生成图找方向，再由代码会话重画成干净的 SVG 或组件。
- 图标要统一、清晰，优先用图标库或 SVG。
- 真实历史照片不能用生成图冒充真实资料。
- 图片生成不适合生成准确中文文字，文字应该由网页 HTML/CSS 排版。

## 老板素材位置占位规则

如果某个位置未来要放老板拍摄的照片、视频、采访、文献资料，但当前还没有真实素材，必须使用明确占位符。

占位符要表达：

- 这里以后可以放真实素材
- 当前只是预留位置
- 不是页面坏了
- 不是假装已有真实内容

推荐占位文案：

- 待放：现场拍摄照片
- 待放：人物采访视频
- 待放：历史文献扫描件
- 待放：VR 导览链接
- 待上传：活动影像资料
- 待补充：资料说明
- 待整理：老照片档案

占位 UI 要做得正式：

- 使用虚线边框或浅色底
- 配一个简洁图标
- 写清楚待放类型
- 可以显示“后台上传后自动展示”
- 不要像报错状态
- 不要写太多解释文字

占位位置包括：

- 首页推荐位
- 视频展播卡片
- 老照片资料库
- 老物件资料库
- 旧文献资料库
- VR 导览入口
- 后台上传区域
- 详情页图片组

重要：生成图可以用作栏目封面和氛围图，但不能用来冒充老板真实拍摄素材。真实素材位置必须标明“待放”或“待上传”。

## Logo 方向

Logo 要体现：

- 向东渠
- 水渠/水流
- 红色文化
- 数字展馆
- 历史档案

推荐做法：

1. 设计会话先生成 3 到 5 个 Logo 概念图。
2. 选一个方向。
3. 代码会话用 SVG 重画，保证清晰、可缩放、无乱码。

Logo 不要：

- 卡通化
- 二次元
- 复杂写实
- 直接把中文文字生成在图片里
- 过度科技感

## 首页 UI 标准

首页第一屏必须有记忆点。

结构建议：

- 顶部导航
- 大幅主视觉
- 标题：向东渠事迹数字馆
- 副标题：一渠清水向东流，五十载精神永传承
- 两个主按钮：进入数字导览、浏览历史资料
- 关键数据：85.81 公里、5 万建设者、1973 年通水、447 处建筑

首页主视觉建议用生成图片：

- 山海之间的人工水渠
- 红色文化展馆氛围
- 历史工程纪实感
- 温暖自然光
- 现代数字展馆界面感
- 不要科幻过头
- 不要人物大特写
- 不要生成文字

## 栏目页 UI 标准

每个栏目页都应该有独立封面图，但整体风格统一。

栏目页结构：

- 栏目头图
- 栏目标题
- 栏目简介
- 分类筛选
- 内容卡片或列表

需要高质量封面图的栏目：

- 数字导览
- 历史资料
- 精神内涵
- 时代价值
- 资讯动态
- 视频展播

## 资料库 UI 标准

资料库要像“数字档案馆”，不是普通图片列表。

视觉关键词：

- 档案盒
- 文献纸张
- 老照片
- 展柜
- 玻璃陈列
- 暖光
- 水利工程纹理

资料卡片字段：

- 封面图
- 标题
- 分类
- 日期
- 简介
- 查看详情

如果资料还没有真实图片，卡片封面使用统一占位样式，并标明：

- 待放：老照片
- 待放：老物件照片
- 待放：文献扫描件

## 后台 UI 标准

后台不能花哨，要像真正可用的内容管理系统。

结构：

- 左侧菜单
- 顶部标题
- 数据统计卡片
- 表格
- 搜索
- 筛选
- 新增按钮
- 编辑入口

后台可以少量使用生成图：

- 登录页背景
- 概览页空状态图
- 上传素材占位图

后台不要大面积使用装饰图，重点是清楚和可信。

后台上传区要明确提示：

- 待上传照片
- 待上传视频
- 待填写 VR 链接
- 待补充文献资料

## 图片资产质量要求

所有生成图片都要满足：

- 有明确用途
- 风格统一
- 适合网页裁切
- 不遮挡文字
- 不依赖图片内文字
- 不冒充真实历史照片
- 文件名清晰
- 记录到 `DESIGN_ASSET_MANIFEST.md`

## 第一批必须精修的视觉资产

优先级从高到低：

1. Logo 概念和最终 SVG
2. 首页主视觉
3. 数字导览封面
4. 历史资料封面
5. 精神内涵封面
6. 时代价值封面
7. 视频展播封面
8. 资讯动态封面
9. 老照片资料库占位图
10. 老物件资料库占位图
11. 旧文献资料库占位图
12. 后台登录页背景

## 给设计会话的硬性要求

设计会话生成图片前，必须先说明：

- 这张图用在哪里
- 为什么需要生成
- 图片风格
- 尺寸比例
- 中文提示词
- 英文提示词
- 保存文件名
- 代码引用路径

生成后必须：

- 保存到项目文件夹
- 更新 `DESIGN_ASSET_MANIFEST.md`
- 不在聊天里堆大量图片
- 只汇报采用图的路径和使用建议

## 判断一张图是否合格

合格标准：

- 一眼能服务“向东渠事迹数字馆”主题
- 放进 UI 后不会显得像素材网站拼图
- 不抢正文内容
- 能提升页面可信度和完成度
- 和其他图片风格一致

不合格表现：

- 太科幻
- 太卡通
- 太像旅游广告
- 太像红色宣传海报但没有数字展馆感
- 图里有乱码文字
- 图中出现不准确历史细节却被当成真实资料
